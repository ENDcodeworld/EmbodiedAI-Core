# 安全策略 | Security Policy

## 支持版本

我们积极为 EmbodiedAI-Core 的最新版本提供安全更新。

| 版本 | 支持状态 |
|------|----------|
| 0.1.x | ✅ 支持中 |
| < 0.1.0 | ❌ 不支持 |

## 报告漏洞

我们非常重视 EmbodiedAI-Core 的安全性。如果你发现任何安全漏洞，请负责任地披露。

### 如何报告

请通过以下方式报告安全漏洞：

- **电子邮件**: embodiedai-security@example.com
- **GitHub**: 使用 [Private Vulnerability Reporting](https://github.com/ENDcodeworld/EmbodiedAI-Core/security/advisories/new)

### 报告内容

为了帮助我们更好地理解和修复漏洞，请尽可能提供以下信息：

- 漏洞类型（例如：缓冲区溢出、SQL 注入、XSS 等）
- 受影响的代码文件或组件
- 完整的复现步骤
- 漏洞的影响（可能被如何利用）
- 如果可能，提供修复建议

### 响应时间

- **确认收到**: 48 小时内
- **初步评估**: 5 个工作日内
- **修复时间表**: 根据严重程度确定
  - 🔴 严重：72 小时内修复
  - 🟠 高：1 周内修复
  - 🟡 中：2 周内修复
  - 🟢 低：下次常规发布

## 安全最佳实践

### 对于用户

1. **保持更新**: 始终使用最新版本的 EmbodiedAI-Core
2. **依赖检查**: 定期运行 `pip audit` 检查依赖漏洞
3. **最小权限**: 以最小必要权限运行机器人控制代码
4. **网络安全**: 确保机器人控制接口不暴露在公共网络

### 对于贡献者

1. **输入验证**: 始终验证外部输入
2. **依赖安全**: 避免引入有已知漏洞的依赖
3. **代码审查**: 所有代码必须经过安全审查
4. **敏感信息**: 不要提交 API 密钥、密码或其他敏感信息
